A/B Testing

In order to see which background color and location of shopping cart worked better, I created two different versions of a cactus shop!

Version A and Version B

Hypothesis for testing

To determine which version was better, we tested two metrics: Time completion and Return Rate.

Time completion

Null Hypothesis: Version A will have an equal or lower time completion rate than Version B

Alternate Hypothesis: Version B will have a higher time completion rate than Version A

The two different features of Version B is that images on Version B change on hover and the cart is placed at the top of the page. The image hover would encourage the user to hover over each image first and then add to cart, increasing the time. The position of the cart forces the user to scroll up to check the contents of the cart, thereby increasing time.

Return Rate

Null Hypothesis: Version A will have an equal or lower return rate than Version B

Alternate Hypothesis: Version B will have a higher return rate than Version A

Version B is more aesthetically pleasing with a neutral combination, better quality images and more features (image hover). This would encourage users to return to the site to explore further.

Data Collection

Half of the data collected was from classmates who were also doing the same project. The other half consisted of my friends and family who did not have a background on this project and only knew the task assigned to them. This could potentially reduce bias.
